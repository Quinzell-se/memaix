## Backlog
New work (feedback, ideas, change requests, bugs) is captured as items in `backlog/`. See
`backlog/TEMPLATE.md`.

Flow: `inbox → triaged → evaluated → approved/rejected → in-dev → done`.
Categories: `backend`, `frontend`, `ux`, `feedback`, `infra`, `security`.
Scoring (1–5): `value` (5 = high), `complexity` (5 = hard), `risk` (5 = high, needs review).

Anyone with project access adds and scores; the backlog is shared. Only the owner moves items to
`approved`/`in-dev`. The AI may suggest category and scores — the human decides.
